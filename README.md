# datalinker（数据处理）

基于 `Tauri + Vue 3 + Element Plus` 的桌面工具，按原 Java 工具迁移实现。  
目标：功能可用优先，不做业务边界扩展。

## 功能概览

当前包含两个页面：
- `数据处理`
- `分组`

## 数据处理页

支持能力：
- 选择 `标准样本路径`、`样本路径`、`解析结果路径`
- 设置 `阈值`
- 添加任务到列表
- 右键菜单：复制选中行 / 删除选中行 / 删除所有行
- 列表中路径通过“选择”按钮更新，支持“查看”打开路径
- 单行执行对比并导出结果

## 导出结果（5 Sheet）

文件名规则：
- `解析结果_<timestamp>.xlsx`

导出包含 5 张 Sheet：
- `亲子鉴定`（汇总）
- `真实性鉴定`（汇总）
- `相同位点`（明细）
- `差异位点`（明细）
- `缺失位点`（明细）

说明：
- 3 张“位点”明细表为按分类拆分后的结果。
- 明细行会包含对比类型、样品编号、标位编号、位点、A/B/C 值等字段。

## 相同/差异/缺失判定规则

### 亲子鉴定（阈值参与）
- 分别对样本 `A/B/C` 与标样 `A/B/C` 做阈值区间命中判断。
- 三项结果求和：
- `0` -> `缺失`
- `>1` -> `相同`
- `=1` -> `差异`

### 真实性鉴定（阈值不参与）
- 若标样 `A/B/C` 全为 `0`，判定为 `缺失`。
- 否则将标样与样本各自 `A/B/C` 排序后比较：
- 完全相等 -> `相同`
- 否则 -> `差异`

## 分组页

支持能力：
- 选择原始 Excel 并解析
- 勾选/全选数据
- 执行分组
- 保存/加载分组配置

## 配置存储

前端配置通过 `tauri-plugin-store` 持久化：
- 文件名：`datalinker.store.json`
- 位置：Tauri `app_data_dir`（随系统与应用标识而定）

## 开发与构建

环境要求：
- Node.js 18+
- pnpm 9+
- Rust stable
- Windows 桌面环境

开发运行：

```bash
pnpm install
pnpm tauri dev
```

构建检查：

```bash
# 前端
pnpm build

# Rust
cd src-tauri
cargo check
```

## 目录结构

```text
datalinker/
  src/            # Vue 前端
  src-tauri/      # Tauri + Rust 后端
  public/         # 静态资源
```
